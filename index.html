Avec la commande : git branch, je constate que je suis sur la branche "main", donc la branche principale

Cette branche à été crée automatiquement lors de la création de se Repository (Repository: Git_Test)

je vais maintenant creer une branche que je vais appeler BRANCH/1. Avec la commande : git branch BRANCH/1, je vais simplement la creer. Si je veux ensuite basculer sur celle-ci, je devrais faire un git chekout BRANCH/1.

Il existe une commande qui permet de faire ces é manipulation en une seule et unique commande. C'est git checkout -b BRANCH/1 !

Je réalise cette commande maintenant

Nous voilà maintenant directement sur la branche BRANCH/1

Je vais faire un git commit -am que je vais nomer "création de la BRANCH/1 + premier Push sur celle-ci avec la commande git push origin BRANCH/1

Il est inutile de push à chaque fois que l'ont crée un "commit". On peu créer le nombre de "commit" que l'on veut, ils apparaitrons tous lors de notre "push". A la fin d'une journée, ou avant la pause déjeuner par exemple.

Pour l'exemple je creerai plusieurs "commit" tout au long du parcours...
